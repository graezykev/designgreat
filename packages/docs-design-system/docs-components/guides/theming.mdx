# Theming

The library includes built-in support for light and dark themes through CSS variables and theme classes.

## Default Theme

The light theme attaches to `:root` by default, so all components use light theme colors without additional configuration.

## Available Themes

- `light` – Default light theme
- `dark` – Dark theme

## Applying a Theme

Theme classes are generated from `@designgreat/lib-web-ui-design-token`. Apply a theme by wrapping your content with the theme class:

```tsx
import { createThemeConfig } from '@designgreat/design-token-support'

const darkTheme = createThemeConfig('dark')

export function App() {
  return <div className={darkTheme.className}>{/* Application content */}</div>
}
```

## Dynamic Theme Switching

For runtime theme switching, use state to manage the active theme:

```tsx
import { useState } from 'react'
import { createThemeConfig } from '@designgreat/design-token-support'

const lightTheme = createThemeConfig('light')
const darkTheme = createThemeConfig('dark')

export function App() {
  const [isDark, setIsDark] = useState(false)
  const currentTheme = isDark ? darkTheme : lightTheme

  return (
    <div className={currentTheme.className}>
      <button onClick={() => setIsDark(!isDark)}>Toggle Theme</button>
      {/* Application content */}
    </div>
  )
}
```

## Theme Classes

You can also directly use the generated theme classes:

```tsx
export function App() {
  return <div className="dg-theme-dark">{/* Application content */}</div>
}
```

Available class names:
- `dg-theme-light` - Light theme (default)
- `dg-theme-dark` - Dark theme

## How Themes Work

Themes work by switching CSS variable values. All components use CSS variables like `var(--dg-color-background-button-default)`, which automatically update when a theme class is applied.

For more details on design tokens and theme architecture, see the [Design Tokens Guide](/docs/design-tokens/overview).

## Troubleshooting

If themes aren't working, see the [Troubleshooting Guide](./troubleshooting.mdx#theme-isnt-applied).
