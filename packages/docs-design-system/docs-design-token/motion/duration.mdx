# Duration

Duration tokens define how long animations and transitions should last. They provide a consistent timing scale that ensures animations feel cohesive across your application.

## Core Duration Scale

| Token | CSS Variable | Value | Use Case |
|-------|--------------|-------|----------|
| <CopyableCode>duration.instant</CopyableCode> | <CopyableCode>--dg-duration-instant</CopyableCode> | 0ms | No animation |
| <CopyableCode>duration.fast</CopyableCode> | <CopyableCode>--dg-duration-fast</CopyableCode> | 100ms | Micro-interactions, hover |
| <CopyableCode bold>duration.normal</CopyableCode> | <CopyableCode bold>--dg-duration-normal</CopyableCode> | 200ms | Standard transitions |
| <CopyableCode>duration.slow</CopyableCode> | <CopyableCode>--dg-duration-slow</CopyableCode> | 300ms | Emphasis, page changes |
| <CopyableCode>duration.slower</CopyableCode> | <CopyableCode>--dg-duration-slower</CopyableCode> | 500ms | Complex animations, modals |

## Usage Guidelines

### ✅ Do

- Use `fast` (100ms) for hover and focus states
- Use `normal` (200ms) as your default for most transitions
- Use `slow` (300ms) for important state changes
- Use `slower` (500ms) sparingly for complex animations

### ❌ Don't

- Use durations longer than 500ms for UI transitions
- Mix different duration values inconsistently
- Use `instant` (0ms) when users expect feedback
- Make loading states feel artificially slow

```javascript
    slower: { value: 500, type: 'time' },
    transition: {
      hover: { value: '{duration.fast}', type: 'time' },
      // ...
    }
  }
}
```
