# Easing

Easing tokens define cubic-bezier curves that control how animations accelerate and decelerate. Natural easing makes animations feel more organic and responsive.

## Easing Functions

| Token                                                   | CSS Variable                                                 | Value                              | Use Case                      |
| ------------------------------------------------------- | ------------------------------------------------------------ | ---------------------------------- | ----------------------------- |
| <CopyableCode>cubic-bezier.linear</CopyableCode>        | <CopyableCode>--dg-cubic-bezier-linear</CopyableCode>        | `cubic-bezier(0, 0, 1, 1)`         | Constant speed, progress bars |
| <CopyableCode>cubic-bezier.ease</CopyableCode>          | <CopyableCode>--dg-cubic-bezier-ease</CopyableCode>          | `cubic-bezier(0.25, 0.1, 0.25, 1)` | General purpose               |
| <CopyableCode>cubic-bezier.ease-in</CopyableCode>       | <CopyableCode>--dg-cubic-bezier-ease-in</CopyableCode>       | `cubic-bezier(0.42, 0, 1, 1)`      | Exits, elements leaving       |
| <CopyableCode bold>cubic-bezier.ease-out</CopyableCode> | <CopyableCode bold>--dg-cubic-bezier-ease-out</CopyableCode> | `cubic-bezier(0, 0, 0.58, 1)`      | Entrances, elements arriving  |
| <CopyableCode>cubic-bezier.ease-in-out</CopyableCode>   | <CopyableCode>--dg-cubic-bezier-ease-in-out</CopyableCode>   | `cubic-bezier(0.42, 0, 0.58, 1)`   | Continuous motion             |

## When to Use Each

### ease-out (Recommended Default)

Best for most UI interactions. Elements arrive quickly and settle smoothly.

```css
/* Button hover, dropdown open, tooltip appear */
.button:hover {
  transform: translateY(-2px);
  transition: transform 100ms var(--dg-cubic-bezier-ease-out);
}
```

### ease-in

Use when elements are leaving or being dismissed.

```css
/* Modal closing, element being removed */
.modal-exit {
  opacity: 0;
  transition: opacity 200ms var(--dg-cubic-bezier-ease-in);
}
```

### ease-in-out

Use for continuous or symmetrical animations.

```css
/* Accordion expand/collapse, tab switching */
.accordion-content {
  transition: height 300ms var(--dg-cubic-bezier-ease-in-out);
}
```

### linear

Use for progress indicators or when constant speed is needed.

```css
/* Progress bar, loading spinner rotation */
.progress-bar {
  transition: width 100ms var(--dg-cubic-bezier-linear);
}
```

## Usage Examples

### TypeScript/JavaScript

```typescript
import { tokens } from '@designgreat/lib-design-token'

// Easing styles
const easing = {
  linear: tokens['cubic-bezier'].linear,
  ease: tokens['cubic-bezier'].ease,
  'ease-in': tokens['cubic-bezier']['ease-in'],
  'ease-out': tokens['cubic-bezier']['ease-out'],
  'ease-in-out': tokens['cubic-bezier']['ease-in-out']
}

// Component example
function AnimatedBox({ easing = 'ease-out', children }) {
  return (
    <div style={{ transitionTimingFunction: `var(--dg-cubic-bezier-${easing})` }}>
      {children}
    </div>
  )
}
```

## Usage Guidelines

### ✅ Do

- Use `ease-out` as your default for most interactions
- Use `ease-in` when elements are exiting or being dismissed
- Use `ease-in-out` for symmetrical animations
- Keep easing consistent within the same interaction

### ❌ Don't

- Use `ease-in` for entrances (feels sluggish)
- Use `linear` for UI transitions (feels mechanical)
- Mix different easings in the same animation
- Create custom curves without clear purpose
