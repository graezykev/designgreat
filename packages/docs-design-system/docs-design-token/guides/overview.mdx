# Design Token Guides

This guide explains how to consume design tokens from `@designgreat/lib-web-ui-design-token` in your application.

## What Are Design Tokens?

Design tokens are the visual design atoms of the design system — specifically, they are named entities that store visual design attributes. They're used in place of hard-coded values to ensure consistency and maintainability across your application.

## Package Exports

`@designgreat/lib-web-ui-design-token` provides four types of outputs:

| Export | Import Path | Use Case |
|--------|-------------|----------|
| **CSS Variables** | `@designgreat/.../css` | Web styling with `var(--dg-*)` |
| **SCSS Variables** | `@designgreat/.../scss/light` | SCSS projects with `$dg-*` |
| **TypeScript APIs** | `@designgreat/lib-web-ui-design-token` | Runtime token access, theme switching |
| **Font Assets** | `@designgreat/.../font` | Self-hosted Roboto font files |

## Key Features

- ✅ **Light & dark themes** — Built-in, ready to use
- ✅ **Type-safe** — Full IntelliSense for all token paths
- ✅ **Framework agnostic** — Works with React, Vue, Next.js, Vite, Remix, etc.
- ✅ **React Native compatible** — JS token objects work directly in StyleSheet
- ✅ **Self-contained fonts** — No CDN required, fonts bundled in package
