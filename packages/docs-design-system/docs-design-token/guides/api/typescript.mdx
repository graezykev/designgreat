# TypeScript Types

The package exports TypeScript types for all tokens, providing full type safety when working with design tokens.

## Available Types

```typescript
import type {
  ThemeName,
  ThemeToken,
  ThemeOverrides
} from '@designgreat/lib-web-ui-design-token'
```

## Type Definitions

### `ThemeName`

Union type of available theme names:

```typescript
type ThemeName = 'light' | 'dark'
```

Use this when you need to specify a theme name:

```typescript
import {
  applyTheme,
  getThemeClassName,
  type ThemeName
} from '@designgreat/lib-web-ui-design-token'

function setTheme(themeName: ThemeName) {
  // Option 1: Use applyTheme utility
  applyTheme(document.body, themeName)

  // Option 2: Use getThemeClassName
  document.body.className = getThemeClassName(themeName)
}
```

### `ThemeToken`

The full token structure for a theme. This represents the complete shape of all design tokens:

```typescript
const tokens: ThemeToken = getThemeTokens('light')

// Access tokens with full type safety
const primaryColor = tokens.color.primary[500]
const spacing = tokens.spacing[11]
const fontSize = tokens.typography.fontSize.body
```

### `ThemeOverrides`

Partial theme structure for `createTheme()`. Use this when you want to override specific tokens:

```typescript
const overrides: ThemeOverrides = {
  color: {
    primary: {
      500: '#FF0000'
    }
  },
  spacing: {
    11: '2rem'
  }
}

const customTheme = createTheme(DEFAULT_THEME, overrides)
```

## Type-Safe Token Access

When working with tokens, TypeScript provides autocomplete and type checking:

```typescript
import { themes } from '@designgreat/lib-web-ui-design-token'

// Access tokens from the themes object with namespace
const lightTokens = themes.light

// ✅ TypeScript knows all available tokens via .dg namespace
const bgColor = lightTokens.dg.color.background.default
const textColor = lightTokens.dg.color.text.default
const spacing = lightTokens.dg.spacing.px16

// ❌ TypeScript will error on invalid paths
const invalid = lightTokens.dg.color.background.nonexistent // Error!
```

## Generic Type Parameters

Some functions accept generic type parameters for additional type safety:

```typescript
import type { ThemeToken } from '@designgreat/lib-web-ui-design-token'

function generateStyles<T extends Partial<ThemeToken>>(tokens: T) {
  // Process tokens with full type safety
  return tokens
}
```

## Type Inference

TypeScript can infer types automatically from the themes object:

```typescript
import { themes } from '@designgreat/lib-web-ui-design-token'

// TypeScript infers the structure
const theme = themes.light

// You get autocomplete for all token properties
console.log(theme.dg.color.primary.default) // ✅ Autocomplete works
console.log(theme.dg.spacing.px16) // ✅ Full type safety
```

## Extending Types

You can extend the base types for custom tokens:

```typescript
import type { ThemeToken } from '@designgreat/lib-web-ui-design-token'

interface CustomThemeToken extends ThemeToken {
  custom: {
    animation: {
      duration: string
    }
  }
}

// Use your extended type
const customTokens: CustomThemeToken = {
  ...baseTokens,
  custom: {
    animation: {
      duration: '300ms'
    }
  }
}
```
