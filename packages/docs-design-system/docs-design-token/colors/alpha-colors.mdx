# Alpha Colors

Alpha colors are semi-transparent versions of accent colors, providing opacity variations for layering, overlays, and subtle visual effects. They are **theme-aware** and automatically adjust based on the current theme.

## Overview

Alpha colors allow you to create depth, layering, and visual hierarchy through transparency. They reference accent colors and apply predefined alpha (opacity) values.

### Key Characteristics

- **Theme-aware**: Values change between light and dark themes
- **Transparency-based**: Each level has a specific opacity value
- **Limited set**: Only 5 alpha levels for simplicity
- **Derived from accents**: Reference accent colors with transparency

## Alpha Color Scales

`color.alpha.neutral` is an alias that references `color.alpha.grey` and can be changed by themes:

```javascript
color.alpha.neutral.1 === color.alpha.grey.1
```

## Design Philosophy

### Why Only 5 Alpha Levels?

The alpha color system is intentionally limited to **5 levels** for:

1. **Simplicity**: Too many options lead to inconsistency
2. **Purpose-driven**: Each level serves a specific use case
3. **Consistency**: Limited choices ensure designers use the same values
4. **Performance**: Fewer tokens mean smaller bundles and faster theming

### Alpha Value Selection

The alpha values are carefully chosen to provide:

- **0.03**: Imperceptible but useful for very subtle backgrounds
- **0.06**: Barely visible, good for hover states
- **0.14**: Clearly visible but subtle, good for dividers
- **0.31**: Medium transparency, good for overlays
- **0.49**: Strong transparency while maintaining layering

## Use Cases

### Level 1 (3%)
- Very subtle hover states
- Nearly invisible backgrounds
- Soft focus indicators

### Level 2 (6%)
- Subtle hover backgrounds
- Card separators
- Inactive state backgrounds

### Level 3 (14%)
- Divider lines
- Border overlays
- Subtle card backgrounds
- Disabled element backgrounds

### Level 4 (31%)
- Modal overlays
- Dropdown shadows
- Selected state backgrounds
- Medium-strength dividers

### Level 5 (49%)
- Strong modal backgrounds
- Prominent overlays
- High-contrast separators
- Loading state overlays

## Usage Examples

### CSS Usage

```css
/* Subtle hover background */
.button:hover {
  background-color: var(--dg-color-alpha-neutral-2);
}

/* Divider line */
.divider {
  border-color: var(--dg-color-alpha-neutral-3);
}

/* Modal overlay */
.modal-backdrop {
  background-color: var(--dg-color-alpha-neutral-5);
```

## Theme Behavior

### Light Theme
In light themes, alpha colors overlay on light backgrounds, creating darker transparent shades.

### Dark Theme
In dark themes, alpha colors overlay on dark backgrounds, creating lighter transparent shades that work with the inverted color scheme.

:::tip
Alpha colors automatically adapt to the theme. The same token will produce appropriate transparency for both light and dark modes.
:::

## Layering and Composition

Alpha colors work best when:

1. **Layered**: Used on top of other backgrounds for depth
2. **Combined**: Mixed with solid colors for rich effects
3. **Subtle**: Applied sparingly for sophistication

```css
/* Example: Layered card with multiple alpha levels */
.layered-card {
  background-color: var(--dg-color-alpha-neutral-1); /* Base layer */
  border: 1px solid var(--dg-color-alpha-neutral-3); /* Border */
  box-shadow: 0 2px 8px var(--dg-color-alpha-neutral-4); /* Shadow */
}

.layered-card:hover {
  background-color: var(--dg-color-alpha-neutral-2); /* Slightly stronger on hover */
}
```

## Usage Guidelines

### ✅ Do

- Use alpha colors for overlays and layering
- Use alpha level 2-3 for hover states
- Use alpha level 3-4 for dividers and borders
- Use alpha level 4-5 for modal backdrops
- Rely on the 5 predefined levels for consistency

### ❌ Don't

- Create custom alpha values outside the 5 levels
- Use alpha colors for primary text (use accent colors instead)
- Overuse alpha colors - they're meant for subtle effects
- Stack too many alpha layers (can cause visual muddiness)

## Accessibility Considerations

:::warning
Alpha colors with low opacity (levels 1-2) may not provide sufficient contrast for text or interactive elements. Always use solid accent colors for:

- Text content
- Interactive button text
- Icon buttons
- Focus indicators (unless combined with solid borders)
:::

### Contrast Testing

When using alpha colors for borders or backgrounds with text:

1. Test in both light and dark themes
2. Verify contrast ratios meet WCAG guidelines
3. Prefer higher alpha levels (4-5) for important UI boundaries

## Technical Details

### File Structure

```
packages/lib-web-ui-design-token/src/tokens/color/alpha/
  ├── grey.js     # Grey alpha levels (1-5)
  └── neutral.js  # Alias to grey alpha
```

### Token Structure

Each alpha color includes an `attributes` field specifying the alpha value:

```javascript
{
  "color.alpha.grey.3": {
    value: "{color.accent.grey.DEFAULT}",
    attributes: { alpha: 0.14 },
    type: "color"
  }
}
```

### CSS Output

Alpha colors are generated as RGBA values in the CSS output:

```css
--dg-color-alpha-neutral-3: rgba(29, 29, 31, 0.14);
```
