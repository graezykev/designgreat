# Semantic Colors

Semantic colors convey specific meanings and system states. They are **theme-aware** and reference specific accent colors that align with universal color associations (red for errors, green for success, etc.).

## Overview

Semantic colors provide consistent visual language for:

- Success and error states
- Warnings and informational messages
- Discovery and new features
- System feedback and notifications

### Key Characteristics

- **Universal meaning**: Colors follow common design conventions
- **Theme-aware**: Automatically adjust for light/dark themes
- **Reference accent colors**: Map to specific saturated accent colors
- **Full gradient support**: Include all emphasis levels from accent colors

## Semantic Color Summary

| Semantic    | Accent Reference | Primary Use Cases                                   |
| ----------- | ---------------- | --------------------------------------------------- |
| **info**    | Blue (Primary)   | Informational messages, help text, neutral feedback |
| **new**     | Purple           | New features, beta labels, discovery callouts       |
| **success** | Green            | Success messages, confirmations, completed states   |
| **warning** | Orange           | Warnings, cautions, non-critical issues             |
| **error**   | Red              | Errors, validation failures, destructive actions    |

## Usage Guidelines

### ‚úÖ Do

- Use semantic colors consistently across your application
- Pair semantic colors with appropriate icons for clarity
- Use `lowest` levels for alert/banner backgrounds
- Use `DEFAULT` or `bold` levels for borders and icons
- Use semantic colors for feedback and system states

### ‚ùå Don't

- Use semantic colors for action hierarchy (use primary/secondary instead)
- Mix semantic meanings (e.g., using error colors for warnings)
- Rely solely on color to convey information (accessibility)
- Use semantic colors for purely decorative purposes
- Override semantic color meanings with custom associations

## Color-Meaning Associations

### Universal Conventions

The semantic color system follows widely accepted conventions:

- üîµ **Blue/Info**: Neutral, trustworthy, informational
- üü£ **Purple/New**: Innovation, discovery, premium
- üü¢ **Green/Success**: Positive, go, complete
- üü† **Orange/Warning**: Caution, attention needed
- üî¥ **Red/Error**: Stop, danger, critical

### Cultural Considerations

:::warning
While these color associations are common in Western design systems, be aware of cultural differences:

- Red may have different meanings in Asian cultures
- Green can signify different things in Middle Eastern contexts
- Always combine colors with icons, text, or other visual cues
  :::

## Accessibility

### Color Alone Is Not Enough

Never rely on color alone to convey semantic meaning:

```jsx
// ‚ùå Bad: Only color indicates error
<input className="border-error" />

// ‚úÖ Good: Color + icon + text
<div className="input-group">
  <input className="border-error" aria-invalid="true" aria-describedby="error-msg" />
  <span id="error-msg" className="error-message">
    {' '}
    <icon name="alert-circle" /> {' Email is required '}
  </span>
</div>
```

### Contrast Requirements

Ensure semantic colors meet WCAG contrast standards:

- Text on colored backgrounds: 4.5:1 minimum
- Large text on colored backgrounds: 3:1 minimum
- UI components and borders: 3:1 minimum

### Testing

Test semantic colors in:

- ‚úÖ Light theme
- ‚úÖ Dark theme
- ‚úÖ Color blindness simulators (protanopia, deuteranopia, tritanopia)
- ‚úÖ High contrast mode

## Complete Token Summary
