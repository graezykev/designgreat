# Accent Colors

Accent colors are derived from base colors and provide a full range of shades for each color. While the gradient levels themselves remain constant, **semantic aliases** automatically switch between themes to provide proper contrast.

## Overview

Accent colors form the backbone of the color system, providing gradient scales that work across different themes and contexts. They are generated from base colors using algorithmic lightness adjustments.

### Key Characteristics

- **Smart theme switching**: Semantic aliases map to different gradient levels per theme
- **Gradient-based**: Each color has multiple shades (1-10 or 1-12)
- **Fixed gradient palette**: All numbered levels (1-12, 1-10) have the same values across themes
- **Semantic aliases**: Named variations that automatically adapt to themes
- **Derived from base**: All accent colors reference base colors

## Color Categories

Accent colors are organized into two main categories based on their theme behavior:

### Fixed Numbered Gradient Colors

These gradient levels have **fixed hex values** that remain constant across all themes. They provide a stable color palette.

## Theme Adaptation

Accent colors use a two-layer architecture for automatic theme adaptation:

1. **Fixed Gradient Levels** (1-12 for grey, 1-10 for saturated): Same hex values in both themes
2. **Semantic Aliases** (bold, DEFAULT, subtle, etc.): Map to different levels per theme

**Example:**

```css
/* Gradient levels - FIXED */
--dg-color-accent-grey-11: #1d1d1f; /* Same in both themes */

/* Semantic aliases - SWITCH */
--dg-color-accent-grey-default: #1d1d1f; /* Light: level 11 */
--dg-color-accent-grey-default: #e8e8e9; /* Dark: level 2 */
```

:::tip Best Practice
**Always use semantic aliases** (like `bold`, `subtle`, `low`, `silent`) for UI elements! They automatically provide appropriate contrast in both themes. Use numbered levels only when you need fixed colors.
:::

## Usage Guidelines

###  Do

- Use semantic aliases (`DEFAULT`, `bold`, `subtle`) instead of numbered levels
- Rely on theme-aware accent colors for consistent theming
- Use neutral colors for text and UI structure
- Use saturated colors for accents and emphasis

###  Don't

- Hardcode gradient level numbers in production code
- Mix base colors with accent colors
- Use accent colors without considering theme contrast

### Theme Switching Via Aliases

While gradient levels themselves don't change, **semantic aliases map to different levels per theme**:

- **Light Theme**: Aliases map to appropriate levels for light backgrounds
  - `bold`  darker levels (e.g., grey.12, blue.10)
  - `subtle`  mid-range levels
  - `silent`/`lowest`  lighter levels (e.g., grey.1, blue.1)

- **Dark Theme**: Aliases map inversely for dark backgrounds
  - `bold`  lighter levels (e.g., grey.1, blue.1)
  - `subtle`  mid-range levels
  - `silent`/`lowest`  darker levels (e.g., grey.12, blue.10)

This **alias mapping inversion** is what provides automatic theme adaptation, not changes to the gradient values themselves.
