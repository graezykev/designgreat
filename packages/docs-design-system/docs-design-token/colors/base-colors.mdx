# Base Colors

Base colors are the foundational color palette of the Design Great system. These colors are **fixed and not changeable by themes**. All other colors in the system are derived from these base colors.

## Overview

Base colors serve as the core reference points for the entire color system. They are carefully selected to provide a consistent visual language across all themes and contexts.

### Key Characteristics

- **Theme-independent**: Base colors remain constant across light and dark themes
- **Reference colors**: All accent, semantic, and other colors derive from these base colors
- **Raw values**: These are the only colors with hardcoded hex values in the system

## Color Categories

The grey color serves as the foundation for all neutral/achromatic colors in the system.

## Color Derivation

Base colors are used to generate:

1. **Accent Colors**: Gradient scales from each base color (see [Accent Colors](./accent-colors))
2. **Primary Colors**: Reference to base blue (see [Primary Colors](./primary-brand-colors))
3. **Semantic Colors**: Map to specific base colors for meaning (see [Semantic Colors](./semantic-colors))
4. **Secondary/Tertiary/Quartus**: Reference to teal, magenta, and lime respectively

## Usage Guidelines

### ✅ Do

- Use base colors as references in token definitions
- Rely on base colors for generating theme-aware accent colors
- Reference base colors in documentation and design specifications

### ❌ Don't

- Use base colors directly in application code
- Modify base color values without updating the entire design system
- Create custom colors that don't derive from base colors

## Technical Details

### Token Structure

Base colors are defined in the design token package at:

```
packages/lib-web-ui-design-token/src/tokens/color/base/
  ├── grey.js       # Neutral base color
  └── saturated.js  # All saturated base colors
```
