# Text Shortcuts

Text color shortcuts provide semantic, ready-to-use color tokens for all text elements in your application. They abstract away the complexity of the underlying color system and make it easy to maintain consistent text styling.

## Overview

Text shortcuts are high-level tokens that map to accent, primary, and semantic colors, providing intuitive names for common text use cases.

### Key Characteristics

- **Semantic naming**: Names describe purpose, not appearance
- **Theme-aware**: Automatically adapt to light/dark themes
- **Consistent hierarchy**: Clear emphasis levels
- **Ready to use**: No need to reference lower-level tokens

const textStyles = {

  success: { color: tokens.color.text.success }

// Component example
function Alert({ type, children }) {
  const textColor = {
    info: tokens.color.text.information,
    success: tokens.color.text.success,
    warning: tokens.color.text.warning,
    error: tokens.color.text.error
  }[type]

  return <div style={{ color: textColor }}>{children}</div>
}

## Text Emphasis Scale

The neutral text colors follow a clear emphasis scale:

```
inverse    ████████████  (for dark backgrounds)
bold       ███████████   (highest emphasis)
DEFAULT    ████████      (standard text)
subtle     ██████        (secondary text)
subtler    ████          (tertiary text)
subtlest   ██            (lowest emphasis)
```

## Usage Guidelines

### ✅ Do

- Use `color.text.DEFAULT` for all body text
- Use `color.text.bold` for headings and high-emphasis text
- Use `color.text.subtle` and `color.text.subtler` for supporting text
- Use `color.text.inverse` for text on dark/colored backgrounds
- Use semantic text colors for feedback messages
- Use `color.text.link` for all interactive text links

### ❌ Don't

- Use multiple text colors in the same paragraph
- Use semantic colors for non-semantic content
- Use `color.text.subtlest` for important information
- Rely solely on color to convey meaning (use icons, weight, size)
- Override text colors with custom values

## Accessibility

### Contrast Requirements

All text color shortcuts meet WCAG AA standards when used on appropriate backgrounds:

| Text Color | Background | Contrast Ratio | WCAG Level |
|------------|------------|----------------|------------|
| `{color.text.bold}` | `{color.background.DEFAULT}` | ≥ 7:1 | AAA |
| `{color.text.DEFAULT}` | `{color.background.DEFAULT}` | ≥ 4.5:1 | AA |
| `{color.text.subtle}` | `{color.background.DEFAULT}` | ≥ 4.5:1 | AA |
| `{color.text.inverse}` | Dark backgrounds | ≥ 4.5:1 | AA |

:::warning
`color.text.subtler` and `color.text.subtlest` may not meet WCAG AA standards for small text. Use these only for:
- Large text (18px+ or 14px+ bold)
- Non-essential content
- Supplementary information
:::

### Best Practices

```css
/* ✅ Good: Sufficient contrast for body text */
.text {
  color: var(--dg-color-text-default);
}

/* ⚠️ Caution: Only for large or non-essential text */
.metadata {
  color: var(--dg-color-text-subtler);
  font-size: 14px;
}

/* ❌ Bad: Insufficient contrast for important text */
.important-message {
  color: var(--dg-color-text-subtlest); /* Don't do this */
}
```

### Testing

Always test text colors:
- ✅ Against your background colors
- ✅ In both light and dark themes
- ✅ With color blindness simulators
- ✅ Using automated contrast checkers
