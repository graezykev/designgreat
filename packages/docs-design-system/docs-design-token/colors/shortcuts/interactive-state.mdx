# Interactive State Shortcuts

Interactive state tokens provide consistent, theme-aware colors for styling interactive elements across different states (hover, active, focus, disabled, etc.).

## Overview

The design system includes pre-defined interaction and state tokens for buttons, inputs, and other interactive components. These tokens automatically adapt between light and dark themes.

### Key States

- **Default**: Resting state of the element
- **Hover**: Mouse cursor over the element
- **Active**: Element is being clicked/pressed
- **Focus**: Element has keyboard focus
- **Disabled**: Element is non-interactive
- **Selected/Activated**: Element is in selected or activated state
- **Visited**: Link has been visited (links only)

## Usage Examples

### TypeScript/JavaScript

```typescript
import { tokens } from '@designgreat/lib-design-token'

// Interactive state styles
const buttonStates = {
  default: {
    backgroundColor: tokens.color.background.button.DEFAULT,
    color: tokens.color.text.inverse
  },
  hover: {
    backgroundColor: tokens.color.background.button.interaction.hover
  },
  active: {
    backgroundColor: tokens.color.background.button.interaction.active
  },
  focus: {
    backgroundColor: tokens.color.background.button.interaction.focus,
    boxShadow: tokens.shadow.focus
  },
  disabled: {
    backgroundColor: tokens.color.background.button.state.disabled,
    opacity: 0.6,
    cursor: 'not-allowed'
  }
}

// Component example
function Button({ disabled, children, ...props }) {
  const [isHovered, setIsHovered] = useState(false)
  const [isActive, setIsActive] = useState(false)

  const getStyles = () => {
    if (disabled) return buttonStates.disabled
    if (isActive) return buttonStates.active
    if (isHovered) return buttonStates.hover
    return buttonStates.default
  }

  return (
    <button
      {...props}
      disabled={disabled}
      onMouseEnter={() => setIsHovered(true)}
      onMouseLeave={() => setIsHovered(false)}
      onMouseDown={() => setIsActive(true)}
      onMouseUp={() => setIsActive(false)}
      style={getStyles()}
    >
      {children}
    </button>
  )
}
```

## Usage Guidelines

### ✅ Do

- Provide clear visual feedback for all interactive states
- Use consistent transition durations across similar elements
- Ensure disabled states are visually distinct
- Always provide accessible focus indicators
- Test all states in both light and dark themes
- Use pointer cursor for interactive elements

### ❌ Don't

- Skip hover or focus states on interactive elements
- Use only color to indicate state (add icons, labels, etc.)
- Make transitions too slow (> 400ms for most interactions)
- Forget to disable pointer events on disabled elements
- Use :active styles without :hover styles
- Ignore keyboard focus states
