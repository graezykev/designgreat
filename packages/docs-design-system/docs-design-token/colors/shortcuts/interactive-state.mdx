# Interactive State Shortcuts

Interactive state shortcuts provide recommended patterns for styling interactive elements across different states (hover, active, focus, disabled, etc.).

## Overview

Interactive state colors are primarily defined at the primary color level. For other elements, apply similar patterns using accent and semantic colors.

### Key States

- **Default**: Resting state of the element
- **Hover**: Mouse cursor over the element
- **Active**: Element is being clicked/pressed
- **Focus**: Element has keyboard focus
- **Disabled**: Element is non-interactive
- **Selected/Activated**: Element is in selected or activated state
- **Visited**: Link has been visited (links only)

## Button State Patterns

### Primary Buttons

```css
/* Using primary colors */
.button-primary {
  background-color: var(--dg-color-primary-default);
  color: white;
}

.button-primary:hover {
  background-color: var(--dg-color-primary-hover-default);
  /* or directly: var(--dg-color-primary-bold) */
}

.button-primary:active {
  background-color: var(--dg-color-primary-active-default);
  /* or directly: var(--dg-color-primary-subtle) */
}

.button-primary:focus {
  outline: 2px solid var(--dg-color-primary-focus-default);
  box-shadow: 0 0 0 4px var(--dg-color-primary-focus-shadow);
}

.button-primary:disabled {
  background-color: var(--dg-color-primary-disabled-default);
  opacity: 0.6;
  cursor: not-allowed;
}
```

## Transition Patterns

### Smooth Transitions

```css

  transition: 
    background-color 200ms ease-in-out,
    color 200ms ease-in-out,
    border-color 200ms ease-in-out;
}

/* Fast transition for immediate feedback */

  transition:
```

## Usage Guidelines

### ✅ Do

- Provide clear visual feedback for all interactive states
- Use consistent transition durations across similar elements
- Ensure disabled states are visually distinct
- Always provide accessible focus indicators
- Test all states in both light and dark themes
- Use pointer cursor for interactive elements

### ❌ Don't

- Skip hover or focus states on interactive elements
- Use only color to indicate state (add icons, labels, etc.)
- Make transitions too slow (> 400ms for most interactions)
- Forget to disable pointer events on disabled elements
- Use :active styles without :hover styles
- Ignore keyboard focus states

## Accessibility

### Focus Management

```css
/* Visible focus for keyboard users */
.interactive:focus-visible {
  outline: 2px solid var(--dg-color-primary-focus-default);
  outline-offset: 2px;
}

/* Hide focus for mouse users (optional) */
.interactive:focus:not(:focus-visible) {
  outline: none;
}
```

### Disabled State

```css
/* Properly disabled element */
.button:disabled {
  background-color: var(--dg-color-primary-disabled-default);
  color: var(--dg-color-text-subtlest);
  cursor: not-allowed;
  pointer-events: none;
  opacity: 0.6;
}

/* Disabled link */
a[aria-disabled="true"] {
  color: var(--dg-color-text-subtlest);
  pointer-events: none;
  text-decoration: none;
}
```

### Reduced Motion

```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

### TypeScript/JavaScript

```typescript
import { tokens } from '@designgreat/lib-design-token'

// Button state styles
const buttonStates = {
  default: {
    backgroundColor: tokens.color.primary.DEFAULT,
    color: 'white'
  },
  hover: {
    backgroundColor: tokens.color.primary.hover.DEFAULT
  },
  active: {
    backgroundColor: tokens.color.primary.active.DEFAULT
  },
  disabled: {
    backgroundColor: tokens.color.primary.disabled.DEFAULT,
    opacity: 0.6,
    cursor: 'not-allowed'
  }
}

// Component with state management
function Button({ disabled, children, ...props }) {
  const [isHovered, setIsHovered] = useState(false)
  const [isActive, setIsActive] = useState(false)

  const getStyles = () => {
    if (disabled) return buttonStates.disabled
    if (isActive) return buttonStates.active
    if (isHovered) return buttonStates.hover
    return buttonStates.default
  }

  return (
    <button
      {...props}
      disabled={disabled}
      onMouseEnter={() => setIsHovered(true)}
      onMouseLeave={() => setIsHovered(false)}
      onMouseDown={() => setIsActive(true)}
      onMouseUp={() => setIsActive(false)}
      style={getStyles()}
    >
      {children}
    </button>
  )
}
```
