# Background Shortcuts

Background color shortcuts provide ready-to-use tokens for backgrounds. These high-level shortcuts abstract the complexity of the underlying color system and ensure consistent styling across your application.

## Background Colors

Background color shortcuts provide semantic, theme-aware colors for various background use cases.

### TypeScript/JavaScript

```typescript
import { tokens } from '@designgreat/lib-design-token'

// Background styles
const backgrounds = {
  page: tokens.color.background.DEFAULT,
  card: tokens.color.background.DEFAULT,
  sidebar: tokens.color.background.bold,
  header: tokens.color.background.bolder,
  success: tokens.color.background.success.DEFAULT,
  error: tokens.color.background.error.DEFAULT
}

// Component example
function Alert({ type, children }) {
  const bgColor = {
    info: tokens.color.background.information.DEFAULT,
    success: tokens.color.background.success.DEFAULT,
    warning: tokens.color.background.warning.DEFAULT,
    error: tokens.color.background.error.DEFAULT
  }[type]

  return (
    <div style={{ 
      backgroundColor: bgColor,
      padding: '16px',
      borderRadius: '4px'
    }}>
      {children}
    </div>
  )
}
```

## Usage Guidelines

### ✅ Do

- Use `color.background.DEFAULT` for main content areas
- Use `color.background.bold/bolder/boldest` for sidebars and navigation
- Use semantic backgrounds for alerts and notifications
- Layer backgrounds for depth and hierarchy

### ❌ Don't

- Use multiple background colors in the same content area without purpose
- Use semantic backgrounds for non-semantic content
- Create custom background colors outside the token system
- Use high-contrast backgrounds without considering text legibility
- Forget to test backgrounds in both light and dark themes

## Accessibility

### Background-Text Contrast

Always ensure sufficient contrast between backgrounds and text:

| Background | Recommended Text Color | Contrast |
|------------|----------------------|----------|
| `{color.accent.neutral.DEFAULT}` | `{color.text.DEFAULT}` | ≥ 4.5:1 ✅ |
| `{color.accent.neutral.bold}` | `{color.text.DEFAULT}` or `{color.text.inverse}` | ≥ 4.5:1 ✅ |
| `{color.background.success.DEFAULT}` | `{color.text.success}` | ≥ 4.5:1 ✅ |
| `{color.background.success.bold}` | `{color.text.success.bold}` (inverse) | ≥ 4.5:1 ✅ |

### Testing

Always test background-text combinations:
- ✅ In light and dark themes
- ✅ With color blindness simulators
- ✅ Using automated contrast checkers
- ✅ With real content and various text sizes

## Background Layering

Create depth by layering different background levels:

```
┌─────────────────────────────────────┐
│ background.DEFAULT (Page)           │
│  ┌─────────────────────────────┐   │
│  │ background.bold (Sidebar)   │   │
│  │                             │   │
│  └─────────────────────────────┘   │

│  │ background.DEFAULT (Card)   │   │
```
