# Border Shortcuts

Border tokens provide consistent, theme-aware borders across your application. The system includes both complete border shorthand tokens and individual border color tokens.

## Overview

The design system includes pre-defined border tokens that combine width, style, and color into ready-to-use values. These tokens automatically adapt between light and dark themes.

### Key Characteristics

- **Theme-aware**: Border colors adapt to light/dark mode
- **Complete shorthand**: Tokens include width, style, and color
- **Semantic support**: Special borders for inputs, dividers, and focus states
- **Consistent widths**: Standardized width scale from thin to thick

### TypeScript/JavaScript

```typescript
import { tokens } from '@designgreat/lib-design-token'

// Border styles
const borders = {
  default: tokens.border.default,
  subtle: tokens.border.subtle,
  bold: tokens.border.bold,
  inputDefault: tokens.border.input.DEFAULT,
  inputHover: tokens.border.input.hover,
  inputFocus: tokens.border.input.focus,
  divider: tokens.border.divider.DEFAULT,
  focus: tokens.border.focus
}

// Component example
function Card({ variant = 'default', children }) {
  const borderStyle = borders[variant]
  return (
    <div style={{ border: `var(--dg-border-${variant})` }}>
      {children}
    </div>
  )
}
```

## Usage Guidelines

### ✅ Do

- Use `--dg-border-default` for most container borders
- Use `--dg-border-input-*` tokens for form elements (including hover states)
- Use `--dg-border-divider-*` for horizontal separators
- Use `--dg-border-focus` alongside `--dg-shadow-focus` for accessibility
- Test border visibility in both light and dark themes

### ❌ Don't

- Create custom border colors outside the token system
- Use hardcoded pixel values for border widths
- Mix border patterns inconsistently
- Use semantic border colors for non-semantic elements
- Forget to consider hover and focus states

## Accessibility

### Focus Indicators

Always provide visible focus indicators:

```css
.interactive-element:focus {
  outline: var(--dg-border-focus);
  outline-offset: 2px;
  box-shadow: var(--dg-shadow-focus);
}
```

### Contrast Requirements

- UI components and borders: Minimum 3:1 contrast ratio
- Consider using `--dg-border-bold` for better visibility
- Test border visibility in both light and dark themes

## Technical Details

### File Structure

```
packages/lib-design-token/src/tokens/
├── border.js                    # Border tokens (shorthand, width, input, divider)
└── color/shortcut/
    ├── border.js                # Border color shortcuts (default, bold, subtle, semantic)
    └── input-interaction.js     # Input border interaction colors (hover, focus)
```

**Key Files:**

- `border.js`: Defines complete border shorthand tokens (width + style + color) including `border.input.hover` which references `{color.border.input.interaction.hover}`
- `color/shortcut/border.js`: Defines base border color tokens (`color.border.DEFAULT`, `color.border.bold`, `color.border.subtle`, etc.)
- `color/shortcut/input-interaction.js`: Defines input-specific border interaction colors (`color.border.input.interaction.hover`, `color.border.input.interaction.focus`)

### Token Reference Chain

The `border.input.hover` token uses a reference chain:

1. **`border.js` (line 80)**: Defines `border.input.hover` with color reference

   ```javascript
   hover: {
     type: 'border',
     value: {
       width: '{border.width.thin}',
       style: 'solid',
       color: '{color.border.input.interaction.hover}'  // ← References color token
     }
   }
   ```

2. **`color/shortcut/input-interaction.js` (line 320)**: Defines the actual color value
   ```javascript
   "input": {
     "interaction": {
       "hover": {
         value: '{color.border.bold}',  // ← Current value (can be customized)
         type: 'color'
       }
     }
   }
   ```

This allows the border color to be customized independently while maintaining the complete border shorthand structure.
