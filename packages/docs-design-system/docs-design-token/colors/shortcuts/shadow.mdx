# Shadow Shortcuts

Shadow tokens provide consistent, theme-aware elevation and depth across your application.

## Overview

The design system includes pre-defined shadow tokens for common use cases. These tokens automatically adapt between light and dark themes.

### Key Characteristics

- **Theme-aware**: Shadow colors adapt to light/dark mode
- **Elevation system**: Semantic levels from `none` to `xl`
- **Focus & Inset**: Special shadows for accessibility and depth effects
- **Performance optimized**: Reasonable blur radius values

## Elevation Shadows

Use elevation shadows to create visual hierarchy and depth:

| Color                                                                                          | Token                   | CSS Variable                 | Use Case                                    |
| ---------------------------------------------------------------------------------------------- | ----------------------- | ---------------------------- | ------------------------------------------- |
| <div className="color-demo" style={{backgroundColor: 'transparent'}}></div>                    | `shadow.elevation.none` | `--dg-shadow-elevation-none` | Flat elements, no elevation                 |
| <div className="color-demo" style={{backgroundColor: 'var(--dg-color-shadow-default)'}}></div> | `shadow.elevation.sm`   | `--dg-shadow-elevation-sm`   | Subtle lift - buttons, cards at rest        |
| <div className="color-demo" style={{backgroundColor: 'var(--dg-color-shadow-default)'}}></div> | `shadow.elevation.md`   | `--dg-shadow-elevation-md`   | Medium elevation - raised cards, dropdowns  |
| <div className="color-demo" style={{backgroundColor: 'var(--dg-color-shadow-default)'}}></div> | `shadow.elevation.lg`   | `--dg-shadow-elevation-lg`   | High elevation - modals, popovers           |
| <div className="color-demo" style={{backgroundColor: 'var(--dg-color-shadow-default)'}}></div> | `shadow.elevation.xl`   | `--dg-shadow-elevation-xl`   | Maximum elevation - floating panels, toasts |

</details>

## Usage Examples

### TypeScript/JavaScript

```typescript
import { tokens } from '@designgreat/lib-design-token'

// Shadow styles
const shadows = {
  elevation: {
    none: tokens.shadow.elevation.none,
    sm: tokens.shadow.elevation.sm,
    md: tokens.shadow.elevation.md,
    lg: tokens.shadow.elevation.lg,
    xl: tokens.shadow.elevation.xl
  },
  focus: tokens.shadow.focus,
  inset: {
    sm: tokens.shadow.inset.sm,
    md: tokens.shadow.inset.md
  }
}

// Component example
function Card({ elevation = 'md', children }) {
  return (
    <div style={{ boxShadow: `var(--dg-shadow-elevation-${elevation})` }}>
      {children}
    </div>
  )
}
```

## Usage Guidelines

### ✅ Do

- Use shadow tokens (`--dg-shadow-elevation-*`) instead of raw values
- Match elevation to component importance (higher = more prominent)
- Use `--dg-shadow-focus` for accessible focus indicators
- Use `--dg-shadow-inset-*` for pressed/recessed states
- Animate shadows with transitions for smooth interactions

### ❌ Don't

- Create custom shadow values outside the token system
- Use elevation inconsistently (e.g., dropdown with higher elevation than modal)
- Use shadows on every element (reduces visual impact)
- Forget to test shadows in both light and dark themes

## Accessibility

### Focus Indicators

Always use the focus shadow token for keyboard navigation:

```css
.interactive-element:focus {
  outline: var(--dg-border-width-medium) solid var(--dg-color-primary-default);
  outline-offset: var(--dg-spacing-space-2xs);
  box-shadow: var(--dg-shadow-focus);
}
```

### High Contrast Mode

Ensure components have visible borders as fallback:

```css
@media (prefers-contrast: high) {
  .card {
    border: var(--dg-border-default);
  }
}
```
