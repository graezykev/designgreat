# Shadow Shortcuts

Shadow tokens provide consistent, theme-aware elevation and depth across your application.

## Overview

The design system includes pre-defined shadow tokens for common use cases. These tokens automatically adapt between light and dark themes.

### Key Characteristics

- **Theme-aware**: Shadow colors adapt to light/dark mode
- **Elevation system**: Semantic levels from `none` to `xl`
- **Focus & Inset**: Special shadows for accessibility and depth effects
- **Performance optimized**: Reasonable blur radius values

## Shadow Tokens

### Elevation Shadows

Use elevation shadows to create visual hierarchy and depth:

| Token | CSS Variable | Use Case |
|-------|--------------|----------|
| `shadow.elevation.none` | `--dg-shadow-elevation-none` | Flat elements, no elevation |
| `shadow.elevation.sm` | `--dg-shadow-elevation-sm` | Subtle lift - buttons, cards at rest |
| `shadow.elevation.md` | `--dg-shadow-elevation-md` | Medium elevation - raised cards, dropdowns |
| `shadow.elevation.lg` | `--dg-shadow-elevation-lg` | High elevation - modals, popovers |
| `shadow.elevation.xl` | `--dg-shadow-elevation-xl` | Maximum elevation - floating panels, toasts |

## Usage Guidelines

### ✅ Do

- Use shadow tokens (`--dg-shadow-elevation-*`) instead of raw values
- Match elevation to component importance (higher = more prominent)
- Use `--dg-shadow-focus` for accessible focus indicators
- Use `--dg-shadow-inset-*` for pressed/recessed states
- Animate shadows with transitions for smooth interactions

### ❌ Don't

- Create custom shadow values outside the token system
- Use elevation inconsistently (e.g., dropdown with higher elevation than modal)
- Use shadows on every element (reduces visual impact)
- Forget to test shadows in both light and dark themes

## Accessibility

### Focus Indicators

Always use the focus shadow token for keyboard navigation:

```css
.interactive-element:focus {
  outline: 2px solid var(--dg-color-primary-default);
  outline-offset: 2px;
  box-shadow: var(--dg-shadow-focus);
}
```

### High Contrast Mode

Ensure components have visible borders as fallback:

```css
@media (prefers-contrast: high) {
  .card {
    border: var(--dg-border-default);
  }
}
```

## Technical Details

### Token Structure

```
shadow/
├── elevation/
│   ├── none      → transparent (no shadow)
│   ├── sm        → subtle lift
│   ├── md        → medium elevation
│   ├── lg        → high elevation
│   └── xl        → maximum elevation
├── focus         → focus ring
└── inset/
    ├── sm        → subtle inset
    └── md        → medium inset
```

### File Structure

```
packages/lib-design-token/src/tokens/
├── shadow.js           # Shadow tokens (elevation, focus, inset)
└── color/shortcut/
    └── shadow.js       # Shadow color shortcut
```

### CSS Variables Reference

```css
/* Elevation shadows */
--dg-shadow-elevation-none: 0 0 0 transparent;
--dg-shadow-elevation-sm: 0 1px 2px var(--dg-color-shadow-default);
--dg-shadow-elevation-md: 0 4px 6px 1px var(--dg-color-shadow-default);
--dg-shadow-elevation-lg: 0 10px 16px 2px var(--dg-color-shadow-default);
--dg-shadow-elevation-xl: 0 20px 24px 4px var(--dg-color-shadow-default);

/* Focus shadow */
--dg-shadow-focus: 0 0 0 2px var(--dg-color-primary-default);

/* Inset shadows */
--dg-shadow-inset-sm: inset 0 1px 2px var(--dg-color-shadow-default);
--dg-shadow-inset-md: inset 0 2px 4px var(--dg-color-shadow-default);

/* Shadow color */
--dg-color-shadow-default: var(--dg-color-alpha-neutral-4);
```

### TypeScript/JavaScript

```typescript
import { tokens } from '@designgreat/lib-design-token'

// Access shadow tokens
const shadows = {
  none: tokens.shadow.elevation.none,
  sm: tokens.shadow.elevation.sm,
  md: tokens.shadow.elevation.md,
  lg: tokens.shadow.elevation.lg,
  xl: tokens.shadow.elevation.xl,
  focus: tokens.shadow.focus,
  insetSm: tokens.shadow.inset.sm,
  insetMd: tokens.shadow.inset.md
}

// Usage in component
function Card({ elevation = 'md', children }) {
  return (
    <div style={{ boxShadow: `var(--dg-shadow-elevation-${elevation})` }}>
      {children}
    </div>
  )
}
```
