# Get Started

Get started contributing to the component library package quickly.

:::info Initial Setup
Complete the [shared setup steps](/contributing#getting-started) first (clone, install, build).
:::

## Package Location

The component library package is located at:

```
packages/lib-web-component/
```

## Development Commands

### Local Development (Storybook)

Start Storybook development server (recommended):

```bash
pnpm --filter @designgreat/lib-web-component storybook
```

Opens at `http://localhost:6006`

**What happens:**
1. Vite-powered Storybook server starts
2. Design tokens are imported directly from `@designgreat/lib-design-token`
3. Tailwind JIT generates utilities on-demand
4. Theme switcher toolbar allows testing light/dark themes

### Other Development Commands

Component development without Storybook:

```bash
pnpm --filter @designgreat/lib-web-component dev
```

## Lint

Lint the component library code:

```bash
pnpm --filter @designgreat/lib-web-component lint
```

## Typecheck

Type check without emitting files:

```bash
pnpm --filter @designgreat/lib-web-component typecheck
```

## Test and Validate

### Run Tests

Run Jest tests:

```bash
pnpm --filter @designgreat/lib-web-component test
```

Watch mode:

```bash
pnpm --filter @designgreat/lib-web-component test -- --watch
```

Coverage:

```bash
pnpm --filter @designgreat/lib-web-component test -- --coverage
```

## Build

Full production build:

```bash
pnpm --filter @designgreat/lib-web-component build
```

This runs the complete pipeline:
1. Clean previous build artifacts
2. Type check and emit `.d.ts` files
3. Bundle components (ESM + CJS) and CSS with Vite
4. Build static Storybook site

**Build artifacts:**
- `dist/` – Published package artifacts
- `storybook-static/` – Deployable Storybook site

### Build Storybook Only

```bash
pnpm --filter @designgreat/lib-web-component storybook:build
```
