# Installation

Install the design token package using pnpm (or npm/yarn):

```bash
pnpm add @designgreat/lib-web-ui-design-token
```

## Package Exports

The package provides the following exports for easy consumption:

### Main Export

```typescript
import {
  // Theme utilities
  getThemeClassName,
  getThemeSelector,
  applyTheme,
  isThemeApplied,
  getOppositeTheme,
  THEME_CLASSES,
  THEME_SELECTORS,
  
  // Token access
  themes,
  light,
  dark,
  createTheme,
  getThemeTokens,
  listThemeNames,
  
  // Types
  type ThemeName,
  type ThemeToken,
  type ThemeOverrides
} from '@designgreat/lib-web-ui-design-token'
```

TypeScript runtime APIs including theme management utilities and token access.

### Font Export

```typescript
import '@designgreat/lib-web-ui-design-token/font'
```

Font-face definitions and font files (self-contained). This import includes both the CSS and the actual font files.

### CSS Theme Exports

#### Light Theme

```css
@import '@designgreat/lib-web-ui-design-token/css/light';
```

Light theme CSS variables.

#### Dark Theme

```css
@import '@designgreat/lib-web-ui-design-token/css/dark';
```

Dark theme CSS variables.

:::info Combined Import
For theme switching support, import the combined CSS file that includes both light (`:root`) and dark (`.dg-theme-dark`) themes:

```css
@import '@designgreat/lib-web-ui-design-token/css';
```

This is the recommended approach for applications that need theme switching.
:::

### Export Summary

| Export | Purpose | Usage |
|--------|---------|-------|
| Main package | TypeScript APIs | Runtime token access |
| `/font` | Font assets | Font-face + font files |
| `/css/light` | Light theme | Single theme CSS |
| `/css/dark` | Dark theme | Single theme CSS |
