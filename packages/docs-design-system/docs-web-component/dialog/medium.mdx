# Medium Dialog

The default size for most dialog use cases.

## Basic Example

export function MediumDialogExample() {
  const [open, setOpen] = useState(false)
  return (
    <>
      <Button onClick={() => setOpen(true)}>Open Medium Dialog</Button>
      <Dialog open={open} onClose={() => setOpen(false)} size="md">
        <DialogHeader>
          <DialogTitle>Edit Profile</DialogTitle>
        </DialogHeader>
        <DialogBody>
          <DialogDescription>
            Update your profile information. Changes will be saved automatically.
          </DialogDescription>
          <div style={{ marginTop: '1rem' }}>
            <p>Form fields would go here...</p>
          </div>
        </DialogBody>
        <DialogFooter>
          <Button variant="outline" onClick={() => setOpen(false)}>
            Cancel
          </Button>
          <Button variant="primary" onClick={() => setOpen(false)}>
            Save Changes
          </Button>
        </DialogFooter>
      </Dialog>
    </>
  )
}

## Usage Guidelines

### When to Use
- Standard forms (2-5 fields)
- Multi-step workflows
- Content that needs moderate space
- Default choice for most dialogs

### Best Practices
- Use for most dialog scenarios
- Balance content density with readability
- Ensure scrollable content has visual indicators
- Keep forms organized with clear labels
