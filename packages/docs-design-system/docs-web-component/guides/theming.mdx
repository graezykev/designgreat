# Theming

The library includes built-in support for light and dark themes through CSS variables and theme classes.

:::tip Full Theme Documentation
For comprehensive theme switching patterns including system preference detection, theme persistence, and all available utilities, see the [Design Token Theme Switching Guide](/design-token/guides/theme-switching).
:::

## Quick Start

The light theme is applied by default. To enable dark theme, wrap your content with the theme class:

```tsx
import { getThemeClassName } from '@designgreat/lib-web-ui-design-token'

export function App() {
  return (
    <div className={getThemeClassName('dark')}>
      {/* Application content */}
    </div>
  )
}
```

## Available Themes

| Theme | Class Name | CSS Selector |
|-------|------------|--------------|
| Light | (none, default) | `:root` |
| Dark | `dg-theme-dark` | `.dg-theme-dark` |

## Dynamic Theme Switching

For runtime theme switching, use React state to manage the active theme:

```tsx
import { useState } from 'react'
import { getThemeClassName, type ThemeName } from '@designgreat/lib-web-ui-design-token'

export function App() {
  const [theme, setTheme] = useState<ThemeName>('light')

  return (
    <div className={getThemeClassName(theme)}>
      <button onClick={() => setTheme(t => t === 'dark' ? 'light' : 'dark')}>
        Toggle Theme
      </button>
      {/* Application content */}
    </div>
  )
}
```

## Troubleshooting

### Theme Isn't Applied

**Problem:** Components always show light theme colors, even when dark theme class is applied.

**Solution:** Use the theme utilities from the design token package:

```tsx
import { getThemeClassName, applyTheme } from '@designgreat/lib-web-ui-design-token'

// Option 1: Use getThemeClassName with a wrapper
<div className={getThemeClassName('dark')}>{/* Your app */}</div>

// Option 2: Apply theme programmatically
applyTheme(document.documentElement, 'dark')
```

**Additional checks:**
- Verify the theme class is applied to an ancestor element of your components
- Check that the CSS file is loaded before components render
- Inspect the element in browser DevTools to confirm the class is present
- Use `isThemeApplied(element, 'dark')` to verify the theme is correctly applied
