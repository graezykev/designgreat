# Installation

:::tip TL;DR
`pnpm add @designgreat/lib-web-component` → import CSS + fonts → use `<Button>`, `<TextInput>`, `<Dialog>`.
:::

:::info Only Need Design Tokens?
If you only need CSS variables and theme utilities without React components, see the [Design Token Installation Guide](/design-token/guides/installation).
:::

## Install

```bash
pnpm add @designgreat/lib-web-component
```

**Peer dependencies:** `react ^18`, `react-dom ^18`

## Setup

```tsx
// In your app entrypoint (main.tsx, _app.tsx, app/layout.tsx)

import '@designgreat/lib-design-token/font'   // Fonts (optional)
import '@designgreat/lib-web-component/dist/lib-web-component.css' // Styles + tokens

import { Button, TextInput, Dialog } from '@designgreat/lib-web-component'

function Example() {
  return (
    <>
      <Button variant="primary">Create account</Button>
      <TextInput label="Email" type="email" placeholder="you@example.com" />
    </>
  )
}
```

Then apply the font to your app:

```css
body {
  font-family: var(--dg-font-family-primary);
}
```

:::tip What's Included
The CSS import includes design tokens, Tailwind utilities, and component styles — but **not fonts**. See [Fonts](./fonts) for font options.
:::

## Tailwind Projects

The library uses [CSS Cascade Layers](https://developer.mozilla.org/en-US/docs/Web/CSS/@layer) for predictable styling:

```
Layer Priority (lowest → highest):
┌─────────────────┐
│ dg-tokens       │  Design tokens (CSS variables)
├─────────────────┤
│ dg-components   │  Component styles (Button, Dialog, etc.)
├─────────────────┤
│ (unlayered)     │  Tailwind utilities — always wins
└─────────────────┘
```

import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';

<TabItem value="v4" label="Tailwind v4" default>

Tailwind v4 uses native CSS layers. Declare layer order explicitly for seamless integration:

```css
/* Declare layer order — library layers + Tailwind layers */
@layer dg-tokens, dg-components, base, components, utilities, app-overrides;

@import '@designgreat/lib-web-component/dist/lib-web-component.css';
@import "tailwindcss";

/* Optional: custom overrides layer */
@layer app-overrides {
  .dg-btn--primary {
    border-radius: 9999px;
  }
}
```

:::tip Why Layer Order Matters in v4
Tailwind v4's `base`, `components`, and `utilities` are real CSS layers. By declaring the full order upfront, you control exactly how library styles, Tailwind styles, and your overrides interact.
:::

</TabItem>
<TabItem value="v3" label="Tailwind v3">

Import the library CSS **before** `@tailwind utilities`:

```css
@tailwind base;
@tailwind components;

/* Library CSS — layered, can be overridden by utilities */
@import '@designgreat/lib-web-component/dist/lib-web-component.css';

@tailwind utilities;
```

**With explicit layer control** (for custom overrides):

```css
/* Declare all layers upfront — order determines priority */
@layer dg-tokens, dg-components, app-overrides;

@tailwind base;
@tailwind components;

@import '@designgreat/lib-web-component/dist/lib-web-component.css';

@tailwind utilities;

/* Your custom overrides in a dedicated layer */
@layer app-overrides {
  .dg-btn--primary {
    border-radius: 9999px;
  }
}
```

</TabItem>

This ensures your utility classes can override library component styles:

```jsx
<Button className="bg-red-500 rounded-none">Custom styled</Button>
```

Add to your content config for tree-shaking:

```js
// tailwind.config.js
module.exports = {
  content: [
    './src/**/*.{js,jsx,ts,tsx}',
    './node_modules/@designgreat/lib-web-component/**/*.js'
  ]
}
```

## Troubleshooting

### Components Don't Have Styles

**Problem:** Components render but have no styling.

**Solution:** Ensure you've imported the CSS:

```tsx
import '@designgreat/lib-web-component/dist/lib-web-component.css'
```

For frameworks with code splitting (Next.js), make sure the import is in `app/layout.tsx` or `pages/_app.tsx`.

### TypeScript Errors

**Problem:** TypeScript can't find type definitions.

**Solution:** Types are included in the package. Run `pnpm install` and verify your `tsconfig.json` has `"moduleResolution": "node"` or `"bundler"`.

### Module Not Found

**Problem:** Build fails with "Cannot find module".

**Solution:** Verify the package is installed:

```bash
pnpm list @designgreat/lib-web-component
```
