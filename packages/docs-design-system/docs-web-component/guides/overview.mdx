# Web Component Guides

This guide explains how to consume the `@designgreat/lib-web-ui` React component library in your application.

:::info Terminology
Despite the name "Web Components", this library provides **React components**, not browser-native [Web Components](https://developer.mozilla.org/en-US/docs/Web/API/Web_components). The name refers to components designed for web applications.
:::

## What Are These Components?

The components in this library are pre-built React UI primitives (Button, TextInput, Dialog) that implement the Design Great design system. They provide consistent styling, accessibility features, and theme support out of the box.

## Package Exports

`@designgreat/lib-web-ui` provides:

| Export | Import Path | What You Get |
|--------|-------------|--------------|
| **React Components** | `@designgreat/lib-web-ui` | Button, TextInput, Dialog |
| **Component Styles** | `@designgreat/.../lib-web-ui.css` | Pre-built styles + design tokens + Tailwind utilities |

:::info Fonts Required Separately
Fonts are **not included** in this package. Import from the design token package:
```tsx
import '@designgreat/lib-design-token/font'
```
See the [Fonts guide](./fonts) for setup details.
:::

## Key Features

- ✅ **Light & dark themes** — Components auto-adapt to theme class
- ✅ **Accessible** — ARIA attributes, keyboard navigation built-in
- ✅ **Type-safe** — Full TypeScript definitions with IntelliSense
- ✅ **Tailwind ready** — Token-aware utility classes included
- ✅ **Framework agnostic** — Works with Next.js, Vite, CRA, Remix, Astro
